# Quickshell Expert

Expert in Quickshell v0.3.0 — a QML-based Wayland shell environment. Refer to the official docs at https://quickshell.org/docs/v0.3.0/ and the QtQuick docs at https://doc.qt.io/qt-6/qtquick-qmlmodule.html.

## Project Structure

Configurations live in `~/.config/quickshell/`. Each named subfolder with a `shell.qml` is a separate config. Run with `quickshell -c <name>` or `quickshell -p /path/to/shell.qml`.

```
~/.config/quickshell/
├── shell.qml                    # default config (if present, subfolders ignored)
└── myconfig/
    ├── shell.qml                # entry point
    ├── .qmlls.ini               # LSP config (auto-generated by Quickshell)
    ├── Bar.qml                  # reusable component
    ├── ClockWidget.qml
    └── Time.qml                 # singleton example
```

## Core Types

### Windows

| Type | Use Case |
|------|----------|
| `PanelWindow` | Bars, panels, widgets, overlays — reserves space on the wlr-layer-shell protocol |
| `FloatingWindow` | Standard desktop windows |
| `PopupWindow` | Popup/menu windows with anchor-based positioning |
| `QsWindow` | Base class for all Quickshell window types |

- `PanelWindow` anchors: `anchors.top`, `anchors.left`, `anchors.right`, `anchors.bottom` (boolean — attaches to screen edges)
- `PanelWindow` exclusion: use `exclusionMode: ExclusionMode.Managed` (default) to reserve space, `ExclusionMode.Normal` or `ExclusionMode.Ignore` for overlays
- `FloatingWindow` has no screen edge anchors — use normal QtQuick positioning

## Idiomatic Code Patterns

### Basic PanelWindow (Multi-Monitor)

```qml
import Quickshell
import QtQuick

Scope {
  id: root
  property string time

  Variants {
    model: Quickshell.screens
    PanelWindow {
      required property var modelData
      screen: modelData
      anchors { top: true; left: true; right: true }
      implicitHeight: 30

      Text {
        anchors.centerIn: parent
        text: root.time
      }
    }
  }

  SystemClock {
    id: clock
    precision: SystemClock.Seconds
  }

  property string time: Qt.formatDateTime(clock.date, "hh:mm:ss AP")
}
```

### Running External Processes

```qml
import Quickshell.Io

Process {
  id: proc
  command: ["sh", "-c", "echo hello"]
  running: true
  stdout: StdioCollector {
    onStreamFinished: console.log(this.text)
  }
}
```

### PopupWindow with Anchors

```qml
import Quickshell

PopupWindow {
  // anchor to a screen edge
  PopupAnchor {
    edges: Edges.Right
    // or use PopupAdjustment for offset
  }
}
```

### Singleton Pattern

```qml
// Time.qml
pragma Singleton
import Quickshell
import QtQuick

Singleton {
  id: root
  readonly property string time: Qt.formatDateTime(clock.date, "hh:mm:ss")
  SystemClock {
    id: clock
    precision: SystemClock.Seconds
  }
}

// Any other file:
// Text { text: Time.time }
```

### Hyprland Integration

```qml
import Quickshell.Hyprland

Hyprland {
  id: hyprland
  // Access monitor/workspace/window state
  // hyprland.monitors, hyprland.activeWorkspace, etc.
}

// Global shortcut
GlobalShortcut {
  shortcut: "SUPER_KEY R"
  onActivated: { /* run command */ }
}
```

### Signal Handling

```qml
Process {
  stdout: StdioCollector {
    // Correct: on<SignalName>
    onStreamFinished: myText.text = this.text
  }
}

// Custom signal in a component:
Item {
  signal clicked()
  MouseArea {
    onClicked: parent.clicked()
  }
}
```

## Size & Positioning Rules

### Core Principle
- **Implicit size** (implicitWidth/implicitHeight) flows **children → parent** — children declare their desired size
- **Actual size** (width/height) flows **parent → children** — parent constrains children
- **NEVER use `childrenRect`** for sizing — it creates binding loops

### Anchor Shorthand
```qml
PanelWindow {
  anchors { top: true; left: true; right: true }
}
// Equivalent to:
// anchors.top: PanelWindow.top
// anchors.left: PanelWindow.left
// anchors.right: PanelWindow.right
```

### Layouts
- Prefer `RowLayout`/`ColumnLayout` over `Row`/`Column` — they support `Layout.fillWidth`, `Layout.fillHeight`, `Layout.preferredWidth`, etc.
- Layouts pixel-align by default; Row/Column do not
- Default layout spacing is 5px

### MarginWrapper
```qml
Item {
  MarginWrapperManager { margin: 5 }
  Rectangle { implicitWidth: 50; implicitHeight: 50 }
}
// Built-in: WrapperItem, WrapperRectangle, WrapperMouseArea
```

## LSP & Editor Setup

- Create empty `.qmlls.ini` next to `shell.qml`; Quickshell manages it
- Known LSP issues: incomplete docs for Quickshell types, PanelWindow not resolved, needs correct brace structure
- Supported editors: Neovim (nvim-lspconfig), Helix (built-in), VSCode (Qt QML extension), Emacs (qml-ts-mode + lsp-mode/eglot)

## Common Errors & Debugging

| Error | Cause | Fix |
|-------|-------|-----|
| `ReferenceError: X is not defined` | Accessed ID outside its component scope | Move to a property on a parent Scope or use a Singleton |
| Zero-sized item warnings | Item or container without implicit size or children | Set `implicitWidth`/`implicitHeight` or add sized children |
| Binding loop | e.g., `width` depends indirectly on itself via `childrenRect` | Use `implicitWidth`/`implicitHeight` properly |
| Panel/bar only on one monitor | Window not replicated per-screen | Wrap in `Variants { model: Quickshell.screens }` |
| Slow clock/battery drain | Process running `date` every second | Use `SystemClock` with appropriate `precision` |

### Debugging Commands
```bash
# Run with verbose output
quickshell -c myconfig -v

# Check for QML errors
quickshell -p path/to/shell.qml 2>&1 | grep -i error
```

## Configuration File Loading

- Searches `$XDG_CONFIG_DIRS/quickshell/` (usually `~/.config/quickshell/`)
- Named subdirectory with `shell.qml` = one config
- If `~/.config/quickshell/shell.qml` exists, subfolders are ignored
- Select with `-c <name>`, run arbitrary path with `-p <path>`
